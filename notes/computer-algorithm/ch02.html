Section A

Chapter Summary

Core concepts distilled from lecture slides — read first.

Insertion Sort Analysis

Insertion Sort is an intuitive sorting algorithm that builds the final sorted array one item at a time. It is particularly efficient for small datasets or nearly sorted arrays.

⭐
Key Properties

Insertion sort is sorted in-place (requires constant extra space) and possesses the stable property (preserves relative order of equal elements).

Case Time Complexity Condition
Best Case O(n) Array is already sorted
Worst Case O(n²) Array is reverse sorted
Average Case O(n²) Random distribution

The Divide-and-Conquer Paradigm

Many useful algorithms are recursive in structure: to solve a given problem, they call themselves recursively one or more times to deal with closely related subproblems. These algorithms typically follow a divide-and-conquer approach.

1
Divide

Break the problem into smaller subproblems.

→
2
Conquer

Solve subproblems recursively (or directly if small).

→
3
Combine

Merge sub-solutions into the final result.

Merge Sort & Recurrence Analysis

Merge Sort is a classic example of divide-and-conquer. It consistently achieves superior performance on large datasets compared to insertion sort.

Divide-and-Conquer Recurrence
T(n) = aT(n/b) + D(n) + C(n)
a = number of subproblems n/b = size of each subproblem D(n) = time to divide C(n) = time to combine
ℹ️
Merge Sort Complexity

For Merge Sort, $T(n) = 2T(n/2) + \Theta(n)$, which results in a total complexity of $\Theta(n \log n)$ in all cases (best, worst, and average).

Section C

Feynman Explanations

Complex concepts explained simply.

🧠

Stable Sort

Why preserving relative order matters.

▼

The core idea: Imagine sorting a list of students by their first name, then sorting them again by their grade. A "stable" sort ensures that if two students have the same grade, they stay in the alphabetical order you already set.

In practice: This is vital when sorting multi-attribute data (like Excel sheets with multiple filters).

The one thing to remember: Stable sorts don't "jumble" items that the current sorting criteria considers equal.

🧠

In-place Sorting

Sorting with zero extra baggage.

▼

The core idea: Think of sorting a shelf of books. If you sort them "in-place," you just swap books directly on the shelf. If you didn't, you'd need a whole second empty shelf to move books onto while you organize them.

In practice: In-place algorithms are memory-efficient because they don't require large auxiliary arrays.

The one thing to remember: In-place means the algorithm uses only a tiny, constant amount of extra memory regardless of input size.

🧠

Divide & Conquer

The "Manager's Approach" to problem solving.

▼

The core idea: A manager doesn't do the whole project. They split the project into tasks (Divide), give tasks to employees (Conquer), and then put the finished pieces together (Combine).

In practice: This turns one $O(n^2)$ problem into many tiny, easy problems that are much faster to handle in total.

The one thing to remember: If you can't solve a big problem, break it into smaller versions of itself until they are trivial to solve.

🧠

Recurrence Equations

The mathematical "Recipe" for time.

▼

The core idea: It's a way to write down the cost of a recursive algorithm. "The time to solve $n$ items equals the time to solve two sets of $n/2$ items, plus the time spent splitting and merging them."

In practice: We use recurrences like $T(n) = 2T(n/2) + n$ to prove exactly how fast an algorithm like Merge Sort really is.

The one thing to remember: A recurrence describes a function in terms of its own values on smaller inputs.

Tier 1 — Fundamental Concepts
1

Define what it means for a sorting algorithm to be "in-place".

An algorithm is in-place if it requires only a constant amount $O(1)$ of additional memory space beyond the input array itself.
2

What is the time complexity of Insertion Sort when the input array is already sorted?

O(n). This is the best-case scenario because the inner loop condition is never met, resulting in only $n-1$ comparisons.
3

List the three steps of the Divide-and-Conquer strategy.

1. Divide the problem into subproblems.
2. Conquer the subproblems recursively.
3. Combine the sub-solutions into the final result.
4

In Merge Sort, if n=8, how many levels are in the recursion tree?

3 levels ($\log_2 8 = 3$), plus the root level.
5

True or False: Merge Sort is a stable sorting algorithm.

True. Merge Sort is stable (assuming the merge step is implemented to prefer the left element in case of ties).
6

Write the recurrence relation that describes the time complexity of Merge Sort.

$T(n) = 2T(n/2) + \Theta(n)$
7

What is the space complexity of Merge Sort?

$O(n)$ extra space is required for the merging process.
8

Why is Insertion Sort named "Insertion" Sort?

Because it works by taking one element at a time from the unsorted part and "inserting" it into its correct position within the sorted part.
9

Compare the best-case performance of Insertion Sort and Merge Sort.

Insertion Sort: $O(n)$ (Best-case).
Merge Sort: $\Theta(n \log n)$ (Best-case).
Insertion Sort is faster on already sorted small inputs.
10

What is the time complexity of merging two sorted arrays of size n/2 into a single sorted array of size n?

$\Theta(n)$
11

Is Merge Sort an in-place algorithm? Explain.

No. Standard Merge Sort requires $O(n)$ auxiliary space to store the merged results before copying them back to the original array.
12

In each iteration of the outer loop of Insertion Sort, what happens to the "key" element?

The key element is compared with elements in the sorted sub-array (to its left) and shifted until its correct insertion point is found.
13

How does Divide-and-Conquer handle the "base case"?

When the subproblem size is small enough (the base case), the algorithm stops recursing and solves the problem directly (e.g., a single element is already sorted).
14

Define the "Stable Sort" property.

A sort is stable if it preserves the relative order of records with equal keys.
15

Solve the recurrence T(n) = 2T(n/2) + cn for a constant c.

$T(n) = \Theta(n \log n)$
Section F

Cheat Sheet

Quick reference for exam preparation.

Complexity Comparison
  • Insertion Sort: $O(n^2)$ worst, $O(n)$ best.
  • Merge Sort: $\Theta(n \log n)$ all cases.
  • In-place: Insertion (Yes), Merge (No).
  • Stable: Insertion (Yes), Merge (Yes).
D&C Framework
Divide
Split $n$ into $a$ subproblems of size $n/b$.
Conquer
Solve recursively; base case is $n \le c$.
Combine
Merge solutions (cost $C(n)$).
Common Mistakes
  • ⚠️ Assuming Merge Sort is in-place ($O(n)$ space!).
  • ⚠️ Forgetting the best-case $O(n)$ for Insertion Sort.
  • ⚠️ Miscalculating recursion tree depth ($\log_2 n$).
