Section A

Chapter Summary

Core concepts distilled from lecture slides — read first.

Definition and Correctness

An algorithm is any well-defined computational procedure that takes some value, or set of values, as input and produces some value, or set of values, as output. Essentially, it is a tool for solving a well-specified computational problem.

✅
Correctness: An algorithm is said to be correct if, for every input instance, it halts with the correct output. An incorrect algorithm might not halt at all on some input instances, or it might halt with an answer other than the desired one.

Algorithm Efficiency

Efficiency is primarily measured in terms of Time (speed) and Space (memory usage). While many problems have efficient solutions, others do not yet have an efficient (polynomial-time) solution known to exist. These are often referred to as "hard" problems.

Donald Knuth is credited with founding the field of analysis of algorithms, while Niklaus Wirth famously stated: Algorithms + Data Structures = Programs.

Algorithms and Data Structures

A Data Structure is a way to store and organize data to facilitate access and modifications. One of the basic techniques for improving algorithms is to structure the data appropriately. Usually, the choice of data structure involves a time-space trade-off, where optimizing for speed may require more memory, and vice-versa.

The Theory of NP-Completeness

Understanding NP-completeness is crucial for an algorithm designer. If you can show that a problem is NP-complete, you can spend your time developing an efficient approximation algorithm rather than searching for a non-existent (or at least undiscovered) optimal polynomial-time solution.

Class P

Decision problems solvable in polynomial time (e.g., O(nk)). These are considered tractable.

Class NP

Nondeterministic Polynomial time. Problems whose solutions can be verified in polynomial time given a certificate.

Class NPC

NP-complete. A subset of NP that is at least as hard as any other problem in NP.

Problem Reduction

Reduction is a technique used to show that a new problem A is NP-complete by transforming a known NP-complete problem B into problem A in polynomial time (B ≤p A). If we can solve A efficiently, we can solve B efficiently. Since B is hard, A must also be hard.

📝 The 1979 textbook by Michael Garey and David S. Johnson, "Computers and Intractability," is the seminal guide to the theory of NP-completeness.

Section C

Feynman Explanations

Deep understanding through simplified analogies.

👨‍🍳

What is an Algorithm?

Explaining it like a recipe.

▼ Expand

The core idea: Think of an algorithm as a recipe. You have ingredients (Input), a specific set of steps to follow (The Procedure), and a finished dish (Output). Just as a recipe must be precise so anyone can follow it to get the same result, an algorithm must be well-defined so a computer can execute it.

In practice: Sorting a list of numbers is like organizing spices alphabetically. You follow a rule (like "find the smallest and put it first") until the task is done.

The one thing to remember: An algorithm is a repeatable "how-to" guide for solving a specific data problem.

📚

Time-Space Trade-off

The bookshelf analogy.

▼ Expand

The core idea: Imagine you have 100 books. You can pile them randomly (takes little space, but takes a long time to find one) or build a massive index system with labels for every page (takes lots of space, but you find things instantly).

In practice: In computing, using a "Hash Map" uses extra memory to allow for nearly instant data retrieval, while a simple "Linear Search" uses no extra memory but is much slower.

The one thing to remember: You can often buy speed by spending memory, or save memory by spending time.

🧩

P vs NP

Solving vs. Verifying.

▼ Expand

The core idea: Class P is like an easy Sudoku: you can solve it quickly. Class NP is like a massive, complex jigsaw puzzle: it might be extremely hard to solve, but if someone shows you the finished picture, you can verify that it's correct very quickly.

In practice: Multiplying two large numbers is in P. Finding the prime factors of a huge number is in NP (hard to do, easy to check if someone gives you the factors).

The one thing to remember: P is about finding the answer; NP is about checking the answer.

🔄

Reduction

The "Translator" analogy.

▼ Expand

The core idea: If you know how to bake a Cake (Problem B), and I show you how to turn the task of making a Muffin (Problem A) into "basically baking a small cake," then you can now solve Problem A using your existing knowledge of Problem B.

In practice: We "reduce" a new problem to an old, hard one to prove the new one is also hard. If A is harder than B, and B is already "impossible," then A is also "impossible."

The one thing to remember: Reduction proves hardness by showing one problem can be transformed into another.

Section E

Exercises

Tier 1 · Recall and basic understanding.

Tier 1 — Fundamental Concepts
1

Define a "correct" algorithm.

An algorithm is correct if, for every input instance, it halts with the correct output.
2

Explain the difference between a decision problem and an optimization problem.

A decision problem has a Yes/No answer (e.g., "Is there a path?"), while an optimization problem seeks the best possible value (e.g., "What is the shortest path?"). NP-completeness is formally defined for decision problems.
3

Why is the theory of NP-completeness important for an algorithm designer?

If a problem can be shown to be NP-complete, the designer can stop looking for an exact polynomial-time algorithm and focus on approximation algorithms or heuristics.
4

What is the complexity class of problems solvable in O(n3) time?

Class P (Polynomial time).
5

True or False: If a problem is in P, it is also in NP.

True. Any problem that can be solved in polynomial time can also be verified in polynomial time (by simply solving it).
6

What is a "certificate" in the context of NP?

A certificate is extra information (like a solution candidate) that allows a verification algorithm to prove the answer is "Yes" in polynomial time.
7

Who is credited with founding the field of analysis of algorithms?

Donald Knuth.
8

What does B ≤p A mean in terms of relative hardness?

It means B reduces to A in polynomial time, implying that A is at least as hard as B.
9

Describe the "time-space trade-off" in algorithm design.

It refers to the phenomenon where an algorithm can be made faster by using more memory, or its memory usage can be reduced at the cost of execution time.
10

List the two requirements for a problem A to be NP-complete.

1. A must be in NP.
2. A must be NP-hard (every problem in NP must reduce to A in polynomial time).
11

Name the influential 1979 textbook by Michael Garey and David S. Johnson.

"Computers and Intractability: A Guide to the Theory of NP-Completeness."
12

What is the current status of the "P vs NP" question?

It is one of the most famous unsolved problems in computer science. Most scientists believe P ≠ NP, but it has not been proven.
13

Can an NP-hard problem be outside of NP?

Yes. A problem is NP-hard if it is "at least as hard as NP," but it doesn't have to be verifiable in polynomial time itself.
14

How does the choice of data structure generally affect an algorithm?

The data structure affects how efficiently data can be accessed and modified, which directly impacts the algorithm's time and space complexity.
15

What is the primary goal of using reduction in complexity theory?

To show that a problem is at least as hard as another known hard problem, thereby classifying its complexity.
Section F

Cheat Sheet

Quick reference for exam preparation.

Complexity Classes
P (Polynomial)
Solved in O(nk). Easy problems.
NP (Nondeterministic Poly)
Verified in O(nk). Hard to find, easy to check.
NPC (NP-Complete)
Hardest problems in NP. Solve one, solve all.
NP-Hard
At least as hard as NP. Not necessarily in NP.
Key Equations & Rules
  • Correctness: Halts + Correct Output.
  • Efficiency: Time + Space Complexity.
  • Wirth's Law: Algorithms + DS = Programs.
  • NPC Proof: Show in NP + Reduce known NPC to it.
Common Mistakes
  • ⚠️ Thinking NP means "Not Polynomial" (It means Nondeterministic Polynomial).
  • ⚠️ Assuming P = NP (Unknown status).
  • ⚠️ Forgetting that reduction (B ≤p A) means A is the harder one.
